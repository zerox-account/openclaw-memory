# AGENTS.md - Agent Capital Workspace Rules

## First Run

If `BOOTSTRAP.md` exists, follow initialization, then delete.

## Every Session

Before starting work:

1. **Read SOUL.md** - Who you are (project finance expert)
2. **Read IDENTITY.md** - Financial modeling framework
3. **Read USER.md** - Zero-X context and pipeline
4. **Read memory/YYYY-MM-DD.md** - Today's activities
5. **Read MEMORY.md** - Pricing data and benchmarks

## Modeling Standards

### Required Model Components
- [ ] 20-year cash flow projection (monthly for first 2 years, then annual)
- [ ] Three scenarios: Base, Upside (+20% revenue), Downside (-20% revenue)
- [ ] Sensitivity analysis on 5+ key variables
- [ ] DSCR and LLCR calculations (if debt)
- [ ] IRR and NPV at multiple discount rates
- [ ] Break-even analysis

### Documentation Requirements
- All assumptions listed with sources
- Rationale for key inputs
- Risk factors identified
- Mitigation strategies noted

### Validation Checklist
- [ ] CAPEX breakdown matches vendor quotes
- [ ] OPEX includes all cost categories
- [ ] Revenue projections are conservative
- [ ] Fuel costs include transport and handling
- [ ] Maintenance includes major overhauls
- [ ] Tax and depreciation calculated correctly
- [ ] Working capital requirements included

## Output Formats

### Executive Summary (1 page)
- Project name and location
- Key metrics (IRR, NPV, payback, ranking)
- Go/no-go recommendation
- Top 3 risks and mitigations

### Full Financial Model (Excel/PDF)
- Input assumptions sheet
- Cash flow projection
- Income statement
- Balance sheet
- Sensitivity tables
- Charts and graphs

### Investment Memo (3-5 pages)
- Project overview
- Market analysis
- Technical description
- Financial analysis
- Risk assessment
- Recommendation

## Communication Rules

### With User (Clemens/Zero-X)
- Lead with ranking and recommendation
- Show base case + sensitivity range
- Highlight risks upfront
- Provide actionable next steps

### External Communications (if authorized)
- Banker/lender format for financing
- Investor presentation for equity
- Conservative assumptions for credibility

## Quality Standards

### Model Accuracy
- [ ] All formulas checked
- [ ] Circular references resolved
- [ ] Sensitivities calculate correctly
- [ ] Units consistent throughout

### Commercial Rigor
- [ ] Pricing validated against market
- [ ] Costs include all relevant items
- [ ] Timeline realistic
- [ ] Counterparty credit considered

### Risk Assessment
- [ ] All major risks identified
- [ ] Quantified where possible
- [ ] Mitigation strategies proposed
- [ ] Contingencies included
