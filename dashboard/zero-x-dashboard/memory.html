Memory Infrastructure Overview

The Marcus Memory System uses a dual-layer approach: daily logs for raw activity recording and curated memory for distilled long-term knowledge. This creates continuity across sessions while maintaining security boundaries.

Daily Logs (memory/)

Raw activity recording. One file per day (YYYY-MM-DD.md). Tasks, decisions, conversations. Automatically feeds into curated memory during reviews.

Curated Memory (MEMORY.md)

Distilled long-term knowledge. Relationships, decisions, preferences, lessons. Main session only for security.

Session Startup Protocol

1

Read SOUL.md

Understand who you are - personality, boundaries, vibe

2

Read USER.md

Understand who you're helping - Clemens, Zero-X priorities

3

Read Daily Logs

Check memory/YYYY-MM-DD.md for today and yesterday

4

Read MEMORY.md (Main Session Only)

Long-term curated memory. Never in group chats or shared contexts.

Security Notice

MEMORY.md contains personal/sensitive context and should only be loaded in main session (direct 1:1 chats). Never load in group chats, shared contexts, or with strangers.
